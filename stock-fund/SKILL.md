# 股票 & 基金实时行情查询

通过 akshare 库查询 A 股（沪深京）、ETF/LOF 基金和开放式基金的实时行情。

## 快速开始

运行查询脚本（推荐方式）：

```bash
# 查询单只股票
python scripts/query.py stock 000001

# 按名称搜索
python scripts/query.py stock 平安银行

# 查询 ETF/LOF 基金
python scripts/query.py etf 510300

# 查询开放式基金净值（估算）
python scripts/query.py fund 110022
```

## 数据类型说明

| 类型 | 说明 | 代码示例 | akshare 函数 |
|------|------|---------|-------------|
| A 股 | 沪深京上市公司普通股 | 000001, 600519 | `stock_zh_a_spot_em()` |
| ETF | 交易所交易基金（场内，可实时报价） | 510300, 159915 | `fund_etf_spot_em()` |
| LOF | 上市开放式基金 | 161725 | `fund_lof_spot_em()` |
| 开放式基金 | 场外基金（每日收盘后公布净值） | 110022 | `fund_value_estimation_em()` |

## 判断查询类型

根据用户的描述判断：
- 代码 6 位纯数字、含 "基金" 字样、含 "ETF/LOF/指数" 字样 → 尝试基金
- 含 "股票" 字样、6 位代码以 0/3/6/8/9 开头的 A 股代码 → 查股票
- 不确定时：先尝试股票，再尝试 ETF，最后尝试开放式基金

常见前缀规律（仅参考）：
- 0xxxxx → 深市主板
- 3xxxxx → 创业板
- 6xxxxx → 沪市主板
- 8xxxxx → 北交所
- 51xxxx / 15xxxx → ETF（场内）
- 16xxxx → LOF

## 核心查询逻辑

### 查股票实时行情

```python
import akshare as ak

# 获取全市场 A 股实时行情，再按代码过滤（推荐）
df = ak.stock_zh_a_spot_em()
row = df[df['代码'] == '000001']

# 或使用新浪接口（有频率限制，尽量少用）
df = ak.stock_zh_a_spot()
```

返回字段包含：代码、名称、最新价、涨跌幅、涨跌额、成交量、成交额、振幅、最高、最低、今开、昨收、量比、换手率、市盈率-动态、市净率

### 查 ETF/LOF 实时行情

```python
# ETF（场内，实时更新）
df = ak.fund_etf_spot_em()
row = df[df['代码'] == '510300']

# LOF
df = ak.fund_lof_spot_em()
row = df[df['代码'] == '161725']
```

返回字段包含：代码、名称、最新价、涨跌额、涨跌幅、成交量、成交额、开盘价、昨收价、最高价、最低价

### 查开放式基金净值估算

```python
# 获取全量净值估算（非场内，每日更新，交易日盘中更新估算值）
df = ak.fund_value_estimation_em('全部')
row = df[df['基金代码'] == '110022']
```

注意：列名是**动态的**，包含日期，如 `2026-04-02-估算数据-估算值`，不是固定的 `估算净值`。用脚本 `scripts/query.py fund <代码>` 可自动处理这种情况。

## 输出格式建议

保持**简洁**，不要输出过多无关信息。重点突出：
- **最新价**（或估算净值）
- **涨跌幅**（用 🔴 红色上涨 / 🟢 绿色下跌的惯例——注意 A 股红涨绿跌）
- **成交量/成交额**（流动性参考）
- **更新时间**（如有）

示例输出：
```
平安银行 (000001)
最新价：  11.83 元
涨跌幅：  +1.29% 🔴
今开：    11.71 | 昨收：11.68
最高：    11.89 | 最低：11.70
成交额：  35.6 亿元
```

不必展示持仓明细、历史业绩、近几年涨幅等延伸信息，除非用户明确要求。

## 代码与名称不匹配时的处理

用户给出的代码和名称可能对不上（例如记错了代码，或名字不准确）。这种情况下**同时展示两个基金**：
1. 用户给出的代码对应的实际基金
2. 按用户给出的名称搜索到的基金

示例：用户说「易方达蓝筹精选混合（110022）」，而 110022 实际是易方达消费行业股票，蓝筹精选的正确代码是 005827。应该两个都查并展示，让用户自己确认想要哪个。不要悄悄只查其中一个。

## 按名称搜索

如果用户给出名称而非代码，从全量数据中搜索：

```python
# A 股按名称模糊搜索
df = ak.stock_zh_a_spot_em()
results = df[df['名称'].str.contains(keyword)]

# ETF 按名称搜索
df = ak.fund_etf_spot_em()
results = df[df['名称'].str.contains(keyword)]
```

若搜索到多条结果，展示简短清单让用户确认再展示详情。

## 注意事项

- akshare 数据来源于东方财富、新浪等免费接口，**非交易时间**仍可查到昨收价等静态数据
- `stock_zh_a_spot_em()` 每次返回全市场 ~5000 条数据，查询单股时速度比较慢，同时最好缓存结果避免重复请求
- **开放式基金**（场外）不能实时行情，只有估算净值（盘中）和官方净值（收盘后 T+1）
- 新浪接口 `stock_zh_a_spot()` 有 IP 频率限制，尽量使用东方财富接口 `stock_zh_a_spot_em()`
